# 백테스트 매매 상세 수익률 표시 설계

## 목적

백테스트의 `종목별 매수/매도 상세 이력`에서 수익률을 소수점 셋째 자리에서 반올림하고, 화면과 CSV 모두 소수점 둘째 자리까지 일관되게 표시한다.

## 현재 동작

- 화면은 `return_pct` 값을 그대로 문자열에 연결하므로 소수 자릿수가 일정하지 않다.
- CSV도 `return_pct` 원본 숫자를 그대로 기록하므로 화면과 동일하게 자릿수가 일정하지 않다.
- 수익률 값은 백테스트 계산 및 손익 판정에도 사용될 수 있으므로 표시 요구가 계산 정밀도에 영향을 주어서는 안 된다.

## 결정

원본 숫자와 계산 로직은 변경하지 않고 출력 경계에서만 서식을 적용한다.

- 화면: 유효한 수익률을 반올림해 항상 두 자리로 표시하고 `%`를 붙인다.
  - 양수: `+12.35%`
  - 음수: `-7.11%`
  - 0 또는 한 자리 소수: `+0.00%`, `+1.20%`
  - 값 없음: `-`
- CSV: 같은 반올림 규칙으로 항상 두 자리 숫자를 기록한다.
  - 양수 부호와 `%`는 붙이지 않는다: `12.35`, `1.20`
  - 값 없음: 기존처럼 빈칸
- 수익률 색상, 필터, 백테스트 계산값 및 API의 숫자형 `return_pct` 계약은 유지한다.

## 고려한 대안

1. **출력 경계에서 각각 포맷팅 — 채택**
   - 계산 정밀도와 API 계약을 보존하면서 화면과 CSV 요구를 충족한다.
2. 백엔드 `return_pct` 자체를 두 자리로 반올림
   - 구현은 단순하지만 원본 정밀도가 사라지고 다른 소비자에게 영향을 줄 수 있어 제외한다.
3. `return_pct_display` 필드를 API에 추가
   - 표시값을 공유할 수 있지만 이번 범위에 새 API 필드를 도입할 필요가 없어 제외한다.

## 변경 범위

- `app.py`의 상세 이력 화면 수익률 포맷터
- `app.py`의 백테스트 CSV 상세 이력 수익률 출력
- 화면 및 CSV 회귀 테스트

README와 백테스트 엔진 계산 로직은 변경하지 않는다.

## 검증

다음 표시 계약을 자동 테스트한다.

| 입력 | 화면 | CSV |
| --- | --- | --- |
| `12.345` | `+12.35%` | `12.35` |
| `-7.105` | `-7.11%` | `-7.11` |
| `1.2` | `+1.20%` | `1.20` |
| `0` | `+0.00%` | `0.00` |
| `None` | `-` | 빈칸 |

변경 후 관련 Flask 테스트, 전체 테스트, Python 컴파일 검사와 Ruff를 실행한다.

## 완료 조건

- 화면과 CSV가 모두 소수점 둘째 자리까지 표시한다.
- 소수점 셋째 자리에서 일반적인 반올림 결과가 나온다.
- 백테스트 결과의 원본 수익률 숫자와 계산 로직은 변하지 않는다.
- 기존 테스트와 새 회귀 테스트가 모두 통과한다.
