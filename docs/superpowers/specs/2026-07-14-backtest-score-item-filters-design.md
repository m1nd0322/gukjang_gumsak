# 백테스트 종합점수·항목 필터 설계

## 목적

백테스트 대상 종목을 종합점수와 스크리닝 항목으로 직접 제한할 수 있게 한다. 화면과 API가 같은 규칙을 적용하고, 점수 다중 선택과 항목 AND 조건을 명확히 정의해 실행할 때마다 예측 가능한 후보군을 만든다.

## 범위

- 백테스트 화면에 종합점수 `3점`, `2점`, `1점` 다중 선택 필터를 추가한다.
- 백테스트 화면에 `연간실적호전`, `순매수전환`, `국민연금 매수` 다중 선택 필터를 추가한다.
- `/api/backtest/run`이 필터 값을 검증하고 백테스트 작업에 전달한다.
- 백엔드가 스크리닝 결과에 필터를 적용한 뒤 백테스트 종목 코드를 결정한다.
- 백테스트 결과 설정에 실제 적용한 필터를 기록한다.
- README에 필터 사용법, 기본값, 결합 규칙을 반영한다.

## 사용자 동작과 필터 의미

### 종합점수

- `3점`, `2점`, `1점`을 동시에 여러 개 선택할 수 있다.
- 선택한 점수 중 하나와 종목의 `종합점수`가 일치하면 점수 조건을 만족한다(OR).
- 아무 점수도 선택하지 않으면 `1점`, `2점`, `3점` 전체를 대상으로 한다.
- 화면의 최초 기본 선택은 기존 동작인 `2점 이상`을 보존하도록 `3점`과 `2점`으로 한다.

### 항목별 필터

- `연간실적호전`, `순매수전환`, `국민연금 매수`를 동시에 여러 개 선택할 수 있다.
- 선택한 항목이 여러 개이면 종목이 선택 항목을 모두 만족해야 한다(AND).
- 아무 항목도 선택하지 않으면 항목으로 종목을 제한하지 않는다.
- 항목 판정은 스크리닝 결과의 쉼표 구분 `출처` 값을 분리한 뒤 정확히 일치하는 항목으로 비교한다. 부분 문자열 일치는 사용하지 않는다.

### 조건 결합

종목은 유효 점수 조건과 선택된 모든 항목 조건을 동시에 만족해야 백테스트 후보가 된다.

```text
후보 여부 = (종합점수 ∈ 유효 점수 집합)
          AND (선택 항목 집합 ⊆ 종목의 출처 항목 집합)
```

## API 계약

`POST /api/backtest/run` 요청 본문은 기존 설정에 다음 필드를 선택적으로 추가한다.

```json
{
  "period": 12,
  "capital": 10000000,
  "strategy": "equal_weight",
  "slippage": 0.1,
  "commission": 0.015,
  "tax": 0.18,
  "scores": [3, 2],
  "items": ["turnaround", "nps"]
}
```

### 안정 키와 표시명

| API 키 | 스크리닝 `출처` 값 | 화면 표시명 |
| --- | --- | --- |
| `turnaround` | `연간실적호전` | 연간실적호전 |
| `supply` | `순매수전환` | 순매수전환 |
| `nps` | `국민연금 신규/추가매수` | 국민연금 매수 |

### 기본값과 하위 호환성

- `scores` 필드를 생략한 기존 API 요청은 `[3, 2]`로 처리해 기존의 `2점 이상` 동작을 유지한다.
- `scores: []`를 명시하면 전체 점수 `[3, 2, 1]`를 대상으로 한다.
- `items` 필드를 생략하거나 `items: []`로 보내면 항목 제한을 적용하지 않는다.
- 중복 값은 한 번만 적용하며, 내부에서는 정해진 점수·항목 순서로 정규화한다.

### 입력 검증

- `scores`는 배열이어야 하고 각 값은 불리언이 아닌 정수 `1`, `2`, `3` 중 하나여야 한다.
- `items`는 배열이어야 하고 각 값은 `turnaround`, `supply`, `nps` 중 하나여야 한다.
- 잘못된 타입이나 허용하지 않은 값이 있으면 작업을 시작하지 않고 HTTP 400과 이해 가능한 오류 메시지를 반환한다.

## 내부 설계

### 필터 정규화

API 경계에서 요청 필드를 검증하고 중복을 제거한다. 필드 생략과 빈 배열은 의미가 다르므로, `scores`의 생략 여부를 먼저 구분한 뒤 정규화한다.

### 순수 후보 필터

스크리닝 결과 목록, 선택 점수, 선택 항목 키를 입력받아 필터링된 목록을 반환하는 부작용 없는 함수를 둔다. 함수는 입력 순서를 보존하며 다음을 수행한다.

1. 빈 점수 선택을 전체 점수 집합으로 해석한다.
2. 항목 키를 실제 `출처` 값으로 변환한다.
3. 각 결과의 점수 일치 여부를 확인한다.
4. `출처`를 쉼표로 분리하고 공백을 제거해 정확한 항목 집합을 만든다.
5. 모든 필수 항목이 포함된 결과만 반환한다.

이 함수는 Flask 요청이나 주가 데이터 조회와 분리해 64개 조합을 빠르고 결정적으로 검증할 수 있게 한다.

### 백테스트 실행 흐름

```text
백테스트 화면
  -> 선택 점수·항목을 JSON으로 전송
  -> API 입력 검증 및 정규화
  -> 백그라운드 작업에 필터 전달
  -> 최신 스크리닝 실행
  -> 순수 후보 필터 적용
  -> 종목명-코드 매핑 및 가격 데이터 조회
  -> 백테스트 실행
  -> 결과 설정에 적용 필터 기록
```

필터 결과가 비어 있으면 가격 조회나 백테스트를 계속하지 않고 `선택한 필터 조건에 맞는 종목이 없습니다.`라는 실패 상태를 기록한다.

## 화면 설계

- 기존 백테스트 설정 영역에 `종합점수` 체크박스 그룹과 `항목별 필터` 체크박스 그룹을 추가한다.
- 종합점수는 `3점`, `2점`이 최초 선택되고 `1점`은 선택되지 않은 상태로 시작한다.
- 항목은 최초에 모두 선택하지 않은 상태로 시작한다.
- 종합점수 그룹에는 `미선택 시 전체`, 항목 그룹에는 `여러 항목 선택 시 모두 만족` 안내를 표시한다.
- 실행 시 체크된 점수는 숫자 배열, 체크된 항목은 안정 키 배열로 요청한다.
- 기존 고정 문구인 `스크리닝 2점 이상 종목`은 선택 가능한 필터를 설명하는 문구로 바꾼다.

## 오류 처리

- 유효하지 않은 필터 요청은 HTTP 400으로 즉시 거절한다.
- 조건에 맞는 후보가 없는 경우 백그라운드 작업을 실패 상태로 종료하고 원인을 화면에 표시한다.
- 필터 적용 이후의 종목 코드 누락, 가격 데이터 부족, 백테스트 계산 오류는 기존 오류 처리 흐름을 유지한다.

## 검증 전략

### 전체 조합 검증

점수 선택 부분집합 8개와 항목 선택 부분집합 8개의 곱인 64개 조합을 모두 검증한다.

- 점수 조합: `{}`부터 `{1, 2, 3}`까지 8개. 빈 집합은 전체 점수로 해석한다.
- 항목 조합: `{}`부터 세 항목 전체까지 8개. 빈 집합은 항목 제한 없음으로 해석한다.
- 합성 데이터는 세 출처 항목의 비어 있지 않은 모든 부분집합 7개를 포함하고, 각 행의 점수는 포함한 출처 항목 수와 일치시킨다.
- 각 조합에서 기대 후보는 `점수 일치 AND 선택 항목 전체 포함` 공식을 독립적으로 계산해 실제 반환 목록과 비교한다.
- 조건상 결과가 0개인 조합도 정상 결과로 검증한다.

### 경계 및 통합 검증

- 필터 필드 생략 시 기존 `[3, 2]` 기본값이 백그라운드 작업에 전달되는지 확인한다.
- 빈 점수 배열, 다중 점수, 다중 항목, 중복 값을 확인한다.
- 잘못된 배열 타입, 점수, 항목 키가 HTTP 400을 반환하고 작업을 시작하지 않는지 확인한다.
- 화면에 여섯 개 체크박스와 안내 문구가 있고 실행 요청에 `scores`, `items`가 포함되는지 확인한다.
- 기존 테스트 전체와 Python 정적 검사, 실제 앱 스모크 테스트를 실행한다.

## 제외 범위

- 스크리닝의 종합점수 계산 방식과 세 항목의 판정 기준은 변경하지 않는다.
- 일일 리포트의 종목 선택 규칙은 변경하지 않는다.
- DuckDB 스키마와 저장 데이터 형식은 변경하지 않는다.
- 새로운 외부 의존성은 추가하지 않는다.
